# server-template

The minimum Cloudflare Worker that runs the standard [Resident](https://github.com/inanimate-tech/resident) back-end server.

**This is what `resident.inanimate.tech` runs by default.**

It is a websocket server that allows you to send apps and events to connected devices, and it acts as a relay so you can push apps from the command line.

Fork it as a starting point for self-hosting or customising your own server.

Two runtime dependencies (`@inanimate/resident`, `agents`); a `wrangler` / TypeScript toolchain for development.

## Deploy

```bash
npm install
npx wrangler deploy
```

When deployed the worker will be live at `<name>.<your-cf-account>.workers.dev`.

(The worker name is set in `wrangler.jsonc` and is currently `resident-server-template`.)

## Push to a device

A device using the Resident sandbox connects via websocket to `wss://<host>/devices/<id>` and stays connected.

By default the host is `resident.inanimate.tech`. Change it in the firmware to your own worker url.

Use the server as a relay to push apps to a connected device:

```bash
curl -X POST https://<your-worker>/devices/<id>/send \
  -H 'Content-Type: application/json' \
  -d '{"type":"app","code":"function on_tick(ctx) screen.text(10,10,\"hi\") screen.flip() end"}'
```

## Customise

The relay logic lives in `@inanimate/resident/cloudflare`. To extend behaviour, subclass `DeviceAgent`:

```ts
import { DeviceAgent } from "@inanimate/resident/cloudflare"

export class MyAgent extends DeviceAgent {
  async onRequest(req: Request): Promise<Response> {
    const url = new URL(req.url)
    if (url.pathname.endsWith("/register") && req.method === "POST") {
      return Response.json({ timezone: "Europe/London" })
    }
    return super.onRequest(req)
  }
}
```

Then export `MyAgent` instead of `DeviceAgent` from `src/worker.ts`, and update the `class_name` in `wrangler.jsonc`'s `durable_objects.bindings` to match. See [the package docs](https://github.com/inanimate-tech/resident-js#extending-deviceagent) for `onConnect` / `onMessage` / the `relay` flag.

For example, you could push a different app to your device's Resident sandbox for every hour of the day. Or push a single app whenever the device connects and then push events in response to a user trigger.

## Layout

```
src/worker.ts       ← 15-line worker entry
wrangler.jsonc      ← Durable Object binding + migration
package.json        ← two runtime deps
tsconfig.json       ← strict TS
env.d.ts            ← generated by `npm run types`
```
